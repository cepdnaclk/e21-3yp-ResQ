Firmware Architecture: Real-Time Compression Detection

ðŸ“… February 3, 2026 Firmware

What We Did

  • Designed firmware architecture for real-time ADC reading and sensor data processing
  • Implemented compression event detection using pressure threshold + hysteresis
  • Developed algorithms for: compression count (events), rate calculation (cpm), depth estimation (mm from Hall sensor)
  • Added pause detection (time without compressions) and placement drift tracking (left/right/up/down)

Challenges

  • Event Debouncing: Initial compression detection triggered multiple false events. Added debounce window (50ms).
  • Rate Calculation: Smoothing required to avoid jitter in live cpm display; implemented rolling average.
  • Placement Drift: Detecting hand drift required comparing pressure distribution across bladder array.

Next Steps

  • Integrate with actual hardware ADC
  • Calibration routine for pressure-to-force conversion
  • Send computed metrics to dashboard (serial/USB for now)
